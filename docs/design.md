# AgentDesk 设计文档

## 项目概述

AgentDesk 是一个基于 Hermes Agent 的桌面端智能体应用，旨在提供可视化的多智能体管理界面。

### 核心目标

1. **可视化管理**: 通过图形界面管理多个 AI 智能体
2. **实时监控**: 实时查看各智能体的运行状态和输出
3. **任务协调**: 协调多个智能体之间的工作
4. **本地优先**: 所有数据本地存储，保护隐私

### 技术选型

| 组件 | 技术栈 | 选择理由 |
|------|--------|----------|
| 前端框架 | React + Vite | 组件化开发，热更新快 |
| 桌面框架 | Tauri | 轻量级，跨平台，原生性能 |
| 后端通信 | Hermes API | 与现有 Hermes 生态无缝集成 |
| 动画引擎 | GSAP | 专业级动画效果 |
| 样式方案 | CSS Variables | 主题定制灵活 |

## 功能模块

### 1. 对话模块 (Chat)

**功能**:
- 与 AI 助手实时对话
- 支持文件上传和图片识别
- 对话历史记录
- 流式响应（计划中）

**技术实现**:
- 使用 `useHermes` Hook 管理 API 连接
- 通过代理服务器解决 CORS 问题
- 支持共享 CLI 会话

### 2. 任务管理模块 (Tasks)

**功能**:
- 创建、编辑、删除任务
- 任务状态管理（待处理/进行中/已完成）
- 任务优先级设置
- 任务进度跟踪

**技术实现**:
- 本地状态管理（后续可扩展为数据库）
- 拖拽排序（计划中）
- 任务提醒（计划中）

### 3. 文件管理模块 (Files)

**功能**:
- 文件上传（拖拽/选择）
- 文件预览
- 文件分类管理
- 批量操作

**技术实现**:
- HTML5 File API
- 拖拽事件处理
- 文件类型识别

### 4. 看板模块 (Dashboard)

**功能**:
- 实时显示所有智能体状态
- 智能体输出日志
- 任务状态监控
- 自动刷新

**技术实现**:
- 定时轮询 API
- tmux 会话读取
- 状态检测算法

## 用户界面设计

### 设计原则

1. **简洁明了**: 界面元素清晰，操作直观
2. **响应式**: 适应不同屏幕尺寸
3. **主题统一**: 深色主题，减少视觉疲劳
4. **动画流畅**: 使用 GSAP 实现平滑过渡

### 颜色系统

```css
:root {
  --bg-primary: #1a1a2e;      /* 主背景 */

  --text-secondary: #a0a0a0;  /* 次文字 */

  --success: #00ff88;         /* 成功色 */
  --warning: #ffaa00;         /* 警告色 */

}
```

## 数据流

### 对话流程

```
用户输入 → React 组件 → useHermes Hook → 代理服务器 → Hermes API
    ↓
响应显示 ← 状态更新 ← JSON 解析 ← API 响应 ← 代理转发
```

### 看板数据流

```
定时器触发 → fetch('/api/dashboard') → 代理服务器
    ↓
tmux 会话读取 → 状态检测 → JSON 序列化 → 前端渲染
```

## 安全考虑

1. **API 密钥**: 通过代理服务器管理，不暴露给前端
2. **CORS**: 代理服务器处理跨域问题
3. **本地存储**: 所有数据本地存储，不上传云端
4. **权限控制**: 基于 API Key 的认证机制

## 性能优化

1. **虚拟滚动**: 大量日志时使用虚拟滚动
2. **防抖处理**: 频繁更新时使用防抖
3. **懒加载**: 组件按需加载
4. **缓存策略**: 合理使用缓存减少请求
