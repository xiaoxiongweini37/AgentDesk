# 会话历史功能设计

## 问题
当前会话按 CLI 会话文件分隔，但用户需要按"一轮对话"管理：
- 新会话需要读取上一轮会话的完整内容
- 类似 session_search 的功能，但更完整

## 方案

### 1. 会话索引
创建一个会话索引文件，记录每个"一轮对话"的元数据：
- 会话 ID
- 标题（用户第一条消息）
- 开始时间
- 结束时间
- 消息数量
- 关键词标签

### 2. 会话内容存储
每个会话存储完整的消息列表：
- user 消息
- assistant 消息
- tool 消息（可选，可能太冗长）

### 3. 会话搜索
支持按关键词搜索会话：
- 搜索标题
- 搜索消息内容
- 按时间范围筛选

### 4. 会话加载
新会话可以加载历史会话：
- 加载完整消息列表
- 作为上下文注入到新会话

## 实现步骤

### 1. 创建会话索引 API
```
GET /api/sessions/index → 返回会话索引
GET /api/sessions/:id/full → 返回完整会话（包括所有消息）
GET /api/sessions/search?q=关键词 → 搜索会话
```

### 2. 修改代理服务器
- 创建会话索引文件
- 支持会话搜索
- 支持加载完整会话

### 3. 修改前端
- 添加会话搜索界面
- 支持加载历史会话
- 显示会话摘要

### 4. 集成到新会话
- 新会话启动时，可以搜索历史会话
- 加载相关历史会话作为上下文
- 帮助用户继续之前的工作
