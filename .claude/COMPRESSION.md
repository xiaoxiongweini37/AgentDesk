# Claude 自动压缩机制

## 工作原理

1. **定期保存** - 每完成一个子任务，自动更新 CLAUDE.md
2. **上下文监控** - 当对话变长时，主动压缩
3. **自动总结** - 生成高质量摘要，保留关键信息
4. **无缝恢复** - 新会话可从摘要中恢复

## 压缩触发条件

- 对话超过 50 轮
- 上下文使用率超过 70%
- 用户说"压缩"或"保存进度"

## 压缩内容

保留：
- 关键决策和结论
- 重要的代码修改
- 未完成的任务
- 文件路径和配置变更

丢弃：
- 中间的思考过程
- 重复的解释
- 临时的调试信息

## 使用方法

### 自动压缩
当上下文快满时，我会自动：
1. 总结当前进度
2. 更新 CLAUDE.md
3. 提示你保存

### 手动压缩
你可以说：
- "保存进度"
- "压缩会话"
- "总结一下"
