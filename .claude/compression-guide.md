# Claude 自动压缩指南

## 压缩策略

### 1. 定期保存（每完成一个子任务）
- 更新 CLAUDE.md 的"当前状态"
- Git commit 当前修改
- 记录关键决策

### 2. 上下文监控
当对话变长时，我会：
- 主动提醒："让我们保存一下进度"
- 生成高质量摘要
- 更新 CLAUDE.md

### 3. 自动压缩触发
当检测到以下情况时：
- 对话超过 50 轮
- 重复解释相同内容
- 用户说"压缩"或"保存"

我会：
1. 总结当前进度
2. 更新 CLAUDE.md
3. 提示你保存

## 压缩内容

### 保留
- ✅ 关键决策和结论
- ✅ 重要的代码修改
- ✅ 未完成的任务
- ✅ 文件路径和配置变更
- ✅ 错误和解决方案

### 丢弃
- ❌ 中间的思考过程
- ❌ 重复的解释
- ❌ 临时的调试信息
- ❌ 已完成的子任务细节

## 恢复流程

新会话开始时：
1. 读取 CLAUDE.md
2. 查看 git log
3. 从"正在进行的工作"继续

## 示例压缩

**压缩前**（100 条消息）：
```
用户：帮我实现功能 A
助手：好的，让我看看代码...
（中间 50 条调试信息）
助手：功能 A 实现完成了
用户：现在做功能 B
助手：好的...
（中间 40 条讨论）
助手：功能 B 完成了
```

**压缩后**（保存到 CLAUDE.md）：
```markdown
## 最近完成的工作
- 2026-06-16: 实现功能 A (commit: abc123)
- 2026-06-16: 实现功能 B (commit: def456)

## 关键决策
- 使用方案 X 因为性能更好
- 文件路径：src/xxx.js

## 待处理任务
- [ ] 功能 C
- [ ] 优化性能
```

## 使用方法

### 自动压缩
当上下文快满时，我会自动：
1. 总结当前进度
2. 更新 CLAUDE.md
3. 提示你保存

### 手动压缩
你可以说：
- "保存进度"
- "压缩会话"
- "总结一下"
- "更新 CLAUDE.md"

### 强制压缩
如果上下文已经满了：
1. 新开会话
2. 说"继续上次的工作"
3. 我会读取 CLAUDE.md 并恢复上下文
